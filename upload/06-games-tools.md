# 06. 게임 탭 — 공동체 운영 도구 (추첨/순번/팀 나누기)

## 목표

오락보다는 마을 회의·행사·봉사활동에서 순서를 정하거나 팀을 나누는 실용적 도구. 누구나 규칙 설명 없이 바로
사용할 수 있는 단순한 UI. 결과는 채팅방에 메시지로 자동 게시되어 전원이 동시에 확인한다.

## 제공 기능 목록

| 기능 | 설명 |
|---|---|
| 사다리타기 | 참가자 선택 → 시작 버튼 |
| 룰렛 돌리기 | 참가자/항목 자동 불러오기 → 돌리기 버튼 |
| 제비뽑기 | N명 중 랜덤 당첨 |
| 랜덤 번호 추첨 | 범위 지정 후 추첨 |
| 순번 정하기 | 참가자 목록 랜덤 정렬 |
| 팀 자동 나누기 | 참가자 목록 + 팀 수 입력 → 자동 배분 |
| 당번 뽑기 | 참가자 중 1인 랜덤 선정 |
| 랜덤 이름 뽑기 | 명단에서 임의 추출 |
| 랜덤 숫자 생성 | 범위 내 숫자 생성 |
| OX 선택 | 단순 O/X 랜덤 |
| 동전 던지기 | 앞/뒤 랜덤 |
| 주사위 굴리기 | 1~6 랜덤 (다중 주사위 지원) |
| 가위바위보 | 사용자 vs AI 1회성 대결 |
| 행운의 박스 | 사전 등록된 항목 중 랜덤 오픈 |
| 오늘의 운세 | 가벼운 재미 요소, AI 또는 고정 문구 풀에서 랜덤 |

행사용 확장 기능(동일 UI 패턴 재사용): 경품 추첨, 번호 추첨, 발표 순서 정하기, 음식 담당 정하기, 차량 배정,
자리 배치, 선물 교환 랜덤 추첨, 노래 순서 정하기.

## 화면 구조 (`/app/games`)

- 카드형 그리드로 위 기능 목록을 아이콘+이름으로 나열 (카카오톡 "더보기" 탭과 유사한 느낌)
- 각 기능 탭 → 해당 게임의 실행 패널(모달 또는 전체화면)로 이동

## 참가자 목록 자동 불러오기

- 대부분의 게임은 "참가자"가 필요 (사다리타기, 룰렛, 순번, 팀 나누기, 당번, 이름 뽑기 등)
- 기본값: 소속 공동체(`communities/{id}`)의 멤버 목록(`users` 컬렉션에서 `communityIds` 배열에 해당 id를 포함한
  문서)을 자동으로 불러와 체크박스 형태로 미리 채워둠
- 사용자는 체크박스로 참가자를 추가/제외할 수 있고, 목록에 없는 이름은 직접 텍스트로 추가 가능(비회원 참가자 대응)

## 채팅방 연동 (핵심 요구사항)

- 게임 실행 위치는 두 가지: (1) 게임 탭에서 직접 실행, (2) 채팅방 입력 영역의 게임 버튼(03 문서)에서 실행
- 어느 경로로 실행하든 결과는 항상 해당 공동체의 `messages` 컬렉션에 다음 형태로 저장:

```
{
  type: "game_result",
  gameResultPayload: {
    gameType: "순번정하기" | "룰렛" | ... ,
    title: string,           // 예: "감자캐기 순번"
    resultSummary: string,   // 예: "① 김영희 ② 이순자 ③ 박미숙 ④ 강영자"
    winner: string | null    // 당첨/당번형 게임에서 단일 결과일 경우
  },
  authorUid, authorName, createdAt ...
}
```

- 채팅 UI(03 문서)는 `game_result` 타입을 카드형 말풍선으로 렌더링
- 홈페이지 Live Chat(05 문서)에도 동일하게 노출되어 방문객도 확인 가능

## 예시 렌더링

- "🎲 감자캐기 순번 — ① 김영희 ② 이순자 ③ 박미숙 ④ 강영자"
- "🎯 오늘 청소 당번 — 당첨! 김영희 🎉"

## 구현 시 주의사항

- 추첨 로직은 클라이언트에서 실행해도 무방하나(단순 랜덤), 결과 저장은 반드시 서버 타임스탬프
  (`serverTimestamp()`) 사용해 순서 보장
- UI는 2단계 이내(참가자 선택 → 시작)로 완결되어야 하며, 추가 설정(고급 옵션)이 필요하면 접이식(collapsible)
  영역에 숨겨 기본 화면은 항상 단순하게 유지
- 게임 이력은 별도 조회 화면 없이 채팅 메시지 히스토리로 충분 (별도 "게임 기록" 페이지는 이번 범위에서 구현하지 않음)
