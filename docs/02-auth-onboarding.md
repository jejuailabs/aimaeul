# 02. 인증 및 온보딩 (Google 로그인 / 회원·비회원 분기)

## 목표

가입 과정을 최대한 단순화한다. Google 로그인 한 번으로 가입이 끝나고, 이후에는 자동 로그인 상태를 유지한다.
회원과 비회원은 첫 화면부터 다르게 라우팅된다.

## 첫 진입 분기 로직

- `/` 접속 시 Firebase Auth 세션 확인
  - **비로그인 상태(비회원)** → 대한민국 마을 지도 화면 (`/`, 05 문서 참고)를 그대로 보여준다
  - **로그인 상태 + 소속 공동체 있음(회원)** → `/app/chat` (카카오톡 스타일 채팅 리스트)로 자동 리다이렉트
  - **로그인 상태 + 소속 공동체 없음(가입 직후)** → `/onboarding`으로 리다이렉트

## Google 로그인 흐름

1. 지도 화면 또는 임의 화면 우상단 "로그인" 버튼 → `/login`
2. `/login`에서 Firebase Auth `signInWithPopup(GoogleAuthProvider)` (모바일 웹에서는 `signInWithRedirect` 사용 권장)
3. 로그인 성공 시 `users/{uid}` 문서 존재 여부 확인
   - 없으면 신규 생성 (`displayName`, `photoURL`은 Google 프로필에서 채움, `communityIds: []`)
   - `/onboarding`으로 이동
4. 이미 `communityIds`가 있으면 바로 `/app/chat`로 이동
5. 이후 세션은 Firebase Auth의 지속 로그인(persistence: local)을 사용해 앱 재실행 시 재로그인 절차 없음

## 온보딩 (`/onboarding`)

두 가지 경로 중 하나를 사용자가 선택한다. 카카오톡 UI 톤(큰 버튼 2개)을 유지.

1. **지도에서 마을 선택**: 대한민국 지도(05 문서의 지도 컴포넌트 재사용)에서 마커를 탭해 공동체 선택
2. **초대코드 입력**: 큰 입력창 하나 + "참여하기" 버튼. `communities` 컬렉션에서 `inviteCode` 일치 문서 탐색

선택/입력 완료 시:
- `users/{uid}.communityIds`에 `communityId` 추가
- 비공개 커뮤니티(`isPublic: false`)인 경우 초대코드 검증 필수, 공개 커뮤니티는 지도 선택만으로 가입 가능
- 완료 후 `/app/chat`로 이동하며, 해당 커뮤니티의 오픈채팅방에 "OO님이 참여했습니다" 시스템 메시지 자동 생성
  (`messages` 컬렉션에 `type: "system"` 문서 추가)

## 로그아웃 / 다중 공동체

- "내 정보" 탭에 로그아웃 버튼 배치 (`/app/me`)
- 한 사용자가 여러 공동체(예: 부녀회 + 청년회)에 속할 수 있으므로 `/app/chat`는 리스트 형태로 여러 채팅방을 보여줄 수 있어야 함
  (카카오톡 채팅 목록과 동일한 UX)
- 추가 공동체 참여는 `/app/home` 또는 "내 정보"에서 "+ 새 마을 참여하기" 버튼으로 온보딩 플로우 재진입

## 구현 시 주의사항

- 로그인 버튼은 카카오톡 로그인 화면을 흉내내지 않는다(상표권/UI 저작권 이슈) — 대신 "Google로 시작하기" 형태의
  일반적인 소셜 로그인 버튼 UI를 사용하되, 색감/레이아웃은 프로젝트 톤(노란 포인트, 둥근 버튼)에 맞춘다
- 비회원은 어떤 경우에도 쓰기 작업(채팅, 사진 업로드, 게임 실행)을 할 수 없다. 모든 쓰기 버튼은 로그인 유도로 연결
- Firebase Auth 세션이 없는 상태에서 `/app/**` 경로 접근 시 미들웨어(`middleware.ts`)로 `/login`으로 리다이렉트
