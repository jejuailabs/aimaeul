# 08. AI 데이터 플랫폼 — 자동 편집 / 일일신문 / 검색 / 스토리 / 타임라인 / 히스토리 / 공개 API

## 목표

이 서비스는 단순 메신저가 아니라 "AI가 읽을 수 있는 디지털 마을 아카이브"다. 모든 공개 콘텐츠(게시글, 사진,
채팅, 공지, 행사)는 사람에게 보여지는 동시에 AI가 활용하기 좋은 구조화된 데이터(JSON/Markdown + 메타데이터)로
자동 축적되어야 한다. Phase 1~3(01~07 문서)에서 쌓인 데이터를 원천으로 사용한다.

## 8-1. AI 자동 편집 (일일 요약)

- 매일 자정(Cloud Scheduler + Cloud Function, 또는 Vercel Cron) 배치로 실행
- 대상 데이터: 해당 날짜의 `messages`, `photos`, `events`, `game_result` 메시지
- Anthropic API(Claude)에 해당 데이터를 프롬프트로 전달해 자연어 요약 생성
  (예: "오늘 봉성리 부녀회는 오전 환경정화 활동을 시작으로 감자 수확을 진행했으며...")
- 결과를 `communities/{id}/dailyDigests/{yyyy-mm-dd}` 문서에 저장 (01 문서 스키마 참고)

## 8-2. AI 일일신문 (`/village/[communityId]/news/[date]`)

- `dailyDigests` 문서를 렌더링하는 공개 페이지
- 구성 요소: 오늘의 사진(대표 사진 N장), 오늘의 행사, 오늘 많이 나온 단어(키워드 태그), 오늘의 대화 요약,
  내일 일정 미리보기
- 마을 홈페이지(05 문서) 상단 또는 별도 섹션에 "오늘의 소식" 링크로 노출

## 8-3. AI 월간 마을 기록

- 월 1회 배치로 해당 월의 `photos`, `events`, `messages`, 신규 `users` 가입 수를 집계
- 집계 항목: 사진 수, 행사 건수, 대화 건수, 신규회원 수, 가장 많이 등장한 장소(EXIF GPS 역지오코딩 클러스터링),
  가장 많이 등장한 음식/키워드(AI 태깅 기반), 가장 활발했던 회원
- 저장 위치: `communities/{id}/monthlyDigests/{yyyy-mm}` (동일 패턴으로 신규 컬렉션 추가)
- 공개 페이지: `/village/[communityId]/monthly/[yyyy-mm]`

## 8-4. AI 검색

- 자연어 검색 입력창(회원 앱의 "사진 탭" 상단, 또는 마을 홈페이지)
- 검색 예시: "작년에 김장했던 사진", "감자캐기 사진", "벚꽃이 피었을 때", "철수 아버님이 올린 사진"
- 구현 방식:
  1. 사진 업로드 시 Claude로 이미지 캡션/태그를 미리 생성해 `photos/{id}.aiTags`, `aiCaption`에 저장해둔다 (04 문서 확장)
  2. 검색 시 사용자 질의를 서버(API Route)에서 받아, `aiTags`/`aiCaption`/EXIF 날짜·위치와 매칭
     (초기 버전은 키워드 매칭 + 날짜 파싱으로 충분, 벡터 임베딩 기반 유사도 검색은 후속 고도화로 TODO 표시)
  3. 결과를 사진 그리드로 반환

## 8-5. AI 스토리 생성

- 사용자가 여러 장(예: 최대 30장)의 사진을 선택 → "AI 스토리 만들기" 버튼
- 선택된 사진들의 EXIF(시간순 정렬), 위치, 업로더 정보를 Claude에 전달해 하나의 이야기 형태 글 생성
  (예: "봉성리 주민들은 새벽부터 감자밭에 모였습니다...")
- 결과는 텍스트로 표시하고, 저장/공유(마을 홈페이지에 게시) 옵션 제공
- "영상 스크립트 생성" 옵션은 텍스트 스크립트 생성까지만 이번 범위에 포함 (실제 영상 렌더링은 범위 밖)

## 8-6. 마을 타임라인

- `/village/[communityId]/timeline` 페이지
- `messages`(사진/게임결과 포함) + `events`를 시간순으로 병합해 하나의 흐름으로 렌더링
- 예: "09:30 김영희님의 감자 수확 사진 / 10:15 박순자님의 채팅 / 11:02 청소당번 룰렛 결과 / 12:30 환경정화 완료"
- 실시간 반영 필요 없음(과거 기록 열람용) — 날짜 선택 UI로 특정일 타임라인 조회

## 8-7. 마을 히스토리

- AI가 중요 이벤트(첫눈, 태풍, 벚꽃 개화, 축제, 운동회, 마을잔치 등)를 자동 식별해 별도 하이라이트로 등록
- 판별 로직: 8-1의 일일 요약 생성 시 Claude에게 "오늘이 마을의 기념할 만한 이벤트인지" 함께 판단하도록 프롬프트에
  포함, `true`로 판단되면 `communities/{id}/history/{eventId}` 문서 생성
- 공개 페이지에서 연도/키워드로 검색 가능 (예: "2026년 태풍 당시")

## 8-8. 공개 API / 정적 파일 제공

공개 커뮤니티(`isPublic: true`)의 데이터는 아래 형태로 외부(지자체, 관광 플랫폼, 검색엔진, AI 서비스, 언론사,
연구기관)에 제공한다.

### REST 엔드포인트 (Next.js Route Handler, `app/api/v1/**`)
- `GET /api/v1/village/{communityId}/feed` — 최근 사진/메시지 피드
- `GET /api/v1/village/{communityId}/news` — 일일신문 목록
- `GET /api/v1/village/{communityId}/photos` — 사진 목록(메타데이터 포함)
- `GET /api/v1/events?communityId=...` — 행사 목록
- `GET /api/v1/village/{communityId}/vacant-houses` — 빈집소개 목록 (07 문서 연동)

### 정적 파일 형태 (ISR로 주기 재생성, AI 크롤러 친화적)
- `/village/{communityId}/feed.json`, `feed.md`
- `/village/{communityId}/events.md`
- `/village/{communityId}/chat-summary.json`

모든 공개 API 응답에는 사람이 올린 원본 콘텐츠 + AI가 생성한 메타데이터(요약, 태그, 키워드)를 함께 포함해,
콘텐츠 하나가 "사람이 보는 페이지"인 동시에 "AI가 소비하는 데이터셋"이 되도록 한다.

## 구현 시 주의사항

- Anthropic API 호출은 반드시 서버(Route Handler/Cloud Function)에서만 수행하고, API 키는 서버 환경변수로 관리
- AI 생성 콘텐츠(요약, 캡션, 스토리)는 생성 실패 시에도 원본 데이터(사진, 채팅)는 정상 노출되어야 함 — AI 레이어는
  부가 기능이지 필수 경로가 아니다
- 배치 작업(일일신문, 월간기록, 히스토리 판별)은 재시도 가능하도록 idempotent하게 설계 (동일 날짜 재실행 시
  덮어쓰기, 중복 생성 방지)
- 공개 API는 요청량이 많을 수 있으므로 캐싱(ISR revalidate 주기, 예: 10분) 적용
