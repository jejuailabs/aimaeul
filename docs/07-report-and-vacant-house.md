# 07. 제보하기 & 우리동네 빈집소개

## 목표

카카오톡 메뉴 방식 그대로, 사진 한 장과 짧은 설명만으로 두 가지 콘텐츠를 생성할 수 있게 한다:
(1) 지역 문제 제보(안전신문고 연계), (2) 빈집 임대 정보(향후 SEO/AEO 및 직거래 확장 자산).

## 7-1. 제보하기

### 화면 (`/app/report` 또는 "내 정보"/"부녀회 홈" 메뉴 내 진입점)
- 사진 첨부(1장 이상, 다중 가능) → EXIF 자동 첨부(04 문서 로직 재사용)
- 상황 설명 텍스트 입력(짧은 텍스트박스)
- "제보하기" 버튼 → `communities/{id}/reports` 문서 생성

### 데이터 모델 (01 문서 스키마 참고, 재기재)
```
{
  reportId, reporterUid, photoStorageUrl, exif, description,
  status: "접수" | "안전신문고 전달완료" | "처리중" | "종료",
  createdAt
}
```

### 처리 흐름
1. 회원이 제보 작성 → `status: "접수"`로 저장
2. 중앙(운영자/관리자 대시보드, 이 문서 범위에서는 간단한 관리자 전용 리스트 화면으로 충분)에서 취합
3. 관리자가 안전신문고 등 외부 시스템으로 실제 송출 처리 후 상태를 `"안전신문고 전달완료"`로 수동 변경
   (안전신문고 자동 연동 API는 이번 범위에 없음 — 상태값 관리까지만 구현, 자동 송출은 TODO로 표시)
4. 제보자는 "내 제보 내역"에서 본인이 올린 건의 상태 변화를 확인 가능

### 구현 시 주의사항
- 제보 내용은 기본적으로 비공개(제보자+관리자만 열람). 공개 커뮤니티라도 `reports`는 `isPublic` 규칙과 무관하게
  항상 비공개로 취급 (Security Rules에서 별도 명시)

## 7-2. 우리동네 빈집소개

### 화면 (`/app/vacant-houses`, 공개 페이지는 `/village/[communityId]/vacant-houses`)
- 등록: 사진 여러 장 + 임대료/보증금(숫자 입력) + 설명 텍스트 + 위치(선택)
- 목록: 카드형 리스트, 사진 썸네일 + 임대료 + 지역명
- 상세: 사진 갤러리 + 상세 설명 + 등록자 연락 방법(회원 프로필 통한 간접 연결 — 전화번호 직접 노출은 지양,
  1차 버전에서는 "채팅으로 문의하기" 버튼으로 해당 등록자와의 커뮤니케이션 경로 제공 정도로 충분)

### 데이터 모델 (01 문서 스키마 참고, 재기재)
```
{
  listingId, posterUid, photos: string[], monthlyRent, deposit,
  description, location, status: "게시중" | "거래완료" | "게시중지",
  createdAt
}
```

### 공개 노출 및 SEO/AEO
- `communities/{id}.isPublic == true`인 경우, 빈집 목록은 비회원에게도 공개 (05 문서 마을 홈페이지에 요약 섹션으로
  노출 가능)
- 각 매물 상세 페이지는 SSR/ISR로 렌더링하여 검색엔진 및 AI 크롤러가 인덱싱 가능하도록 함 (08 문서의 공개 API/정적
  파일 제공 방식과 연계 — `GET /api/v1/village/{id}/vacant-houses` 형태로 노출)
- 향후 확장(직거래 쇼핑몰 연계)은 이번 범위에 포함하지 않으며, 데이터 모델만 그대로 재사용 가능하도록 설계해둔다

### 구현 시 주의사항
- 임대료/보증금은 숫자 필드로 저장하되 "가격 협의" 같은 케이스를 위해 `null` 허용
- 매물 상태(`status`) 변경은 등록자 본인만 가능
- 사진은 04 문서와 동일한 업로드/썸네일 파이프라인 재사용 (별도 구현 금지, 코드 재사용)
